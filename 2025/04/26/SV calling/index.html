通过比较基因组学，从组装好的contig或者reads中鉴定出SV (structure variation) 是进行sv鉴定的两条经典途径。2024年8月，中山大学李淼新团队，基于三代测序数据对53个SV检测的流程进行了评估主流SVcalling流程，并指出，将不同calling得到的vcf文件进行合并可以有效的提升鉴定的精度。

Structural Variation

基因组结构变异（Structural Variations, SVs）是指基因组中长度大于50 bp的大规模变异，主要包括缺失（Deletion）、重复（Duplication）、倒位（Inversion）、插入（Insertion）、易位（Translocation）等

同时，在结果的vcf文件中，有的变异被标注为BND (breakend)，其代表为染色体断裂或易位（Inter-chromosomal Translocation）或染色体内复杂重排（Intra-chromosomal Complex Rearrangement）事件的变异类型，当变异被鉴定为 BND 时，通常表示该变异涉及两个染色体之间的断点连接或同一染色体上不连续断点的重新连接。

注意，svision-pro calling所鉴定到的sv中存在

SVTYPE=INS+idDUP+BND+INS

SVTYPE=INS+idDUP+DEL

SVTYPE=INS+idDUP+INS+idlDUP+DEL

这样的鉴定结果，甚至存在DEL和INS这两种相斥的结果存在，不知道为什么。怀疑是这个方法导致的，本质是将比对结果转化为图像，之后使用GNN神经网络模型进行变异鉴定。

File format

VCF

VCF（Variant Call Format） 是存储基因变异信息（如 SNP、INDEL、结构变异 SV）的标准格式，由**头文件（Header）和数据行（Data Lines）**组成。

  • 核心功能：描述样本相对于参考基因组的变异信息（类型、位置、基因型等）。
  • 适用场景：变异检测结果共享、下游分析（如过滤、注释、统计）。

头文件解析（Header）

头文件以 ## 开头，定义文件格式、来源、参考基因组、变异类型等信息。以下是示例中各部分的详细说明：

(1) 基础信息
(2) 参考基因组（Contig）

定义染色体的名称（ID）和长度（length）：

  • 作用：明确参考基因组的染色体结构，用于坐标定位。
(3) 变异类型（ALT）

描述不同结构变异类型（INS, DEL, DUP, INV, BND）：

  • 关键字段：ID 表示变异类型缩写，Description 为详细解释。
(4) 信息字段（INFO）

定义 INFO 列的字段含义，用于描述变异的属性：

  • 关键参数：
    • ID：字段名称（如 SVTYPE）。
    • Number：值的数量（1 表示单个值，. 表示数量不定）。
    • Type：数据类型（String, Integer, Float, Flag）。
    • Description：字段描述。
(5) 过滤条件（FILTER）

定义 FILTER 列的过滤标签：

  • 用途：标记低质量变异（如 q5 表示质量值低于 5 的变异被过滤）。
(6) 格式字段（FORMAT）

定义 FORMAT 列及样本列的基因型字段格式：

  • 示例：GT:DR:DV 表示每个样本的基因型、参考支持读数和变异支持读数。

数据行解析（Data Lines）

以 #CHROM 开头的行定义列标题，后续每行为一个变异记录：

(1) 列标题
  • 核心列：
    • #CHROM：染色体名称（如 Chr1）。
    • POS：变异起始位置（基于 1 的坐标）。
    • ID：变异标识符（如 cuteSV.INS.0）。
    • REF：参考序列碱基。
    • ALT：替代序列碱基。
    • QUAL：质量分数（Phred 格式，值越高可信度越高）。
    • FILTER：过滤标签（PASS 表示通过过滤）。
    • INFO：变异的附加信息。
    • FORMAT：样本列的格式定义。
    • NULL：样本名称（此处样本名为 NULL）。
(2) 示例数据行
  • 关键字段解析：
    • #CHROM: Chr1（染色体 1）。
    • POS: 5194129（变异起始位置）。
    • ID: cuteSV.INS.0（工具生成的唯一标识符）。
    • REF: T（参考碱基为 T）。
    • ALT: 插入的序列（插入类型，具体序列在 INFO 中描述）。
    • QUAL: 133.7（高质量变异）。
    • FILTER: PASS（通过所有过滤条件）。
    • INFO:
      • PRECISE：标记为精确检测的变异。
      • SVTYPE=INS：变异类型为插入。
      • SVLEN=1827：插入序列长度为 1827 bp。
      • END=5194129：变异结束位置（与起始位置相同，表示点插入）。
      • CIPOS=-287,287：位置置信区间为 [POS-287, POS+287]。
      • RE=18：支持该变异的测序 reads 数。
      • AF=0.8182：等位基因频率（81.82% 的 reads 支持变异）。
    • FORMAT: GT:DR:DV:PL:GQ（样本列字段定义）。
    • 样本数据 1/1:4:18:134,18,0:18:
      • GT=1/1：纯合变异（两个等位基因均为变异）。Genotype，基因型
      • DR=4：支持参考等位的高质量 reads 数。（Reference Read Depth，参考等位支持读数）
      • DV=18：支持变异等位的高质量 reads 数。（Variant Read Depth，变异等位支持读数）
      • PL=134,18,0：三种基因型（0/0, 0/1, 1/1）的 Phred 似然值。（Phred-scaled Genotype Likelihoods，基因型似然值）
      • GQ=18：基因型质量值。（Genotype Quality，基因型质量）

Annotation VCF

数据行例子

  • GT:AD:DP:SAC
    • GT（基因型）：如 0/1（杂合）、1/1（纯合变异）。
    • AD（等位支持读数）：如 0,20 表示参考支持 0 条，变异支持 20 条。
    • DP（总读数深度）：如 20 表示总支持读数为 20。
    • SAC（链特异性读数）：如 0,0,11,9 表示正向参考、反向参考、正向变异、反向变异的读数。
  • 功能筛选：通过 Func.refGene 和 ExonicFunc.refGene 筛选可能具有功能影响的变异（如 exonic 或 stoploss）。
  • 长度分析：统计 SVLEN 分布，比较不同变异类型（INS/DEL/DUP）的长度差异。
  • 基因关联：根据 Gene.refGene 关联已知基因功能，优先研究重要基因（如抗病基因）。
