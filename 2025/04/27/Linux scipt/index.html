Linux脚本是由一系列Shell命令组成的文本文件，通过Shell解释器（如Bash）逐行执行，用于自动化任务、系统管理或批量处理操作。脚本通常以 #!/bin/bash 开头声明解释器，可直接调用系统命令、定义变量和逻辑控制（如循环、条件判断），无需编译，赋予执行权限后通过 ./script.sh 运行，是高效管理系统和简化重复性工作的核心工具。

Heade

shell脚本通常是一个以shebang起始的文本文件

其中#!位于解释器路径之前。/bin/bash是Bash的解释器命令路径。

还有一种常见的写法是#!/bin/bash -ex这里的-e 类似于在第二行写set -e其意义是Exit immediately if a command exits with a non-zero status. ；而-x 的意思是Print commands and their arguments as they are executed.

Command

在 Linux Shell 中，内置命令（Built-in Commands） 是直接集成在 Shell 解释器（如 Bash）中的命令，无需调用外部程序，执行效率更高。

文件与目录操作

  • cd
    切换目录：cd /path 或 cd ~（返回家目录）。
  • pwd
    显示当前工作目录路径。
  • echo
    输出内容：echo "Hello" 或 echo $PATH。echo 加或者不加单双引号都可以打印echo 后面的内容，默认情况下echo在每次调用后会添加一个换行符。
  • read
    读取用户输入：read -p "输入名字: " name。
  • exec
    执行命令并替换当前 Shell 进程：exec ls（执行后退出当前 Shell）

变量与状态操作

  • export
    导出环境变量：export VAR=value。

    同时我们可以在自己home目录下的.bashrc文件添加一些路径，以指定二进制文件或者库文件，让每次shell启动时执行，例如，export PATH=/opt/myapp/bin:$PATH

  • var=value

    var是变量名，value是赋给变量的值。如果value不包含任何空白字符（例如空格），那么它就不需要使用引号进行引用，否则必须使用单引号或双引号。在变量名之前加上$前缀就可以打印出变量的内容。以在echo命令的双引号中引用变量值。

  • unset
    删除变量或函数：unset VAR。

  • readonly
    设为只读变量：readonly PI=3.14。

  • local
    在函数内定义局部变量：local var=123。

文件描述与重定向

在编写脚本的时候会频繁使用标准输入（stdin）、标准输出（stdout）和标准错误（stderr）。

通过内容过滤将输出重定向到文件是我们平日里的基本任务之一。当命令输出文本时，这些输出文本有可能是错误信息，也可能是正常的（非错误的）输出信息。单靠查看输出的文本本身，我们没法区分哪些是正常，哪些是错误。

这个时候可以通过文件描述符来解决问题。

  • 0 标准输入
  • 1 标准输出
  • 2 标准错误

当一个命令发生错误并退回时，它会返回一个非0的退出状态；而当命令成功完成后，它会返回数字0。退出状态可以从特殊变量$? 中获得（在命令执行之后立刻运行echo $?，就可以打印出退出状态）

分别重定向输出和错误可以使用cmd 2>stderr.txt 1>stdout.txt ；重定向到一个文件可以使用cmd 2>&1 output.txt

不查看输出内容或者错误，可以重定向到“垃圾桶”/dev/null 中。

tee 可以在显示输出的同时，将输出内容重定向到一个文件，但是需要注意的是，错误信息是不会保存到重定向的文件中的，tee 只能接受stdout。如果像追加操作而非覆盖的话，需要使用 tee -a 参数。

Debug

可以使用set-x和set+x对脚本进行调试。 透明化脚本的实际执行过程，将隐藏的逻辑错误、变量问题暴露成“可见”的命令行操作，大幅提升排查效率。

  1. set -x
    开启调试模式，执行时会打印每条命令及其展开后的内容（如变量值），方便观察实际执行过程。
  2. set +x
    关闭调试模式，后续命令不再打印调试信息。
  3. 限定在 set -x 和 set +x 之间
    将需要调试的代码块包裹在这两个命令之间，只有这部分代码会输出调试信息，其他部分保持静默。

特殊变量（Special Variables）

  • $0：当前脚本的名称。
  • $1, $2, …$9：脚本或函数的第 1-9 个参数。
  • $#：传递给脚本或函数的参数个数。
  • $@：所有参数的列表（每个参数独立）。
  • $\*：所有参数的列表（合并为一个字符串）。
  • $?：上一个命令的退出状态码（0 表示成功）。
  • $$：当前 Shell 的进程 ID (PID)。
  • $!：最后一个后台进程的 PID。
  • $_：上一个命令的最后一个参数。

参数扩展（Parameter Expansion）

  • ${var}：显式界定变量名（避免歧义）。

  • ${var:-default}：若 var 未定义或为空，使用默认值。

  • ${var#pattern}：删除变量值中匹配 pattern 的最短前缀。

  • ${var##pattern}：删除最长前缀。

  • ${var%pattern}：删除最短后缀。

  • ${var%%pattern}：删除最长后缀。

Run in parallel

  • nohup
    • 不要挂断任务，即把任务提交到服务器上运行
  • &
    • 任务后台运行，通常与nohup连用，这样即使出现了网络问题也正常运行
  • top
    • 实时显示系统中各个进程的资源占用情况，按q退出
  • htop
    • 优化过的top，u可以选择users
  • ps
    • process status列出当前系统中运行的进程列表，一般加上-ef，搭配grep进行搜索，如：ps -ef | grep 用户名
  • kill
    • 杀掉进程 kill -9 PID，第9种命令是指无条件停止

GNU parallel

GNU parallel是一种Shell工具，用于使用一台或多台计算机并行执行作业。作业可以是单个命令，也可以是每一行运行的小脚本.

Download

  • 安装到本地目录（解决权限问题）

  • 之后将安装目录加入到PATH环境变量中。

  • echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

示例：并行压缩多个文件

parallel gzip ::: file1.txt file2.txt file3.txt

输入来自文件列表

echo “file1.txt file2.txt file3.txt” > list.txt
parallel gzip :::: list.txt

示例：并行拼接文件前缀和后缀

parallel echo {1}-{2} ::: A B ::: 1 2

输出：

A-1

A-2

B-1

B-2

使用 4 个并行任务

parallel -j 4 convert {} -resize 800x600 {.}_small.jpg ::: *.jpg


parallel -k ‘sleep {}; echo {}’ ::: 3 1 2

输出顺序为 3, 1, 2（但任务执行顺序为 1, 2, 3）

parallel –halt now,fail=1 ‘grep “error” {}’ ::: *.log


parallel cp {} /backup/{/} ::: *.log # 复制文件到备份目录并保留文件名

输入文件 data.csv 每行格式：name,age,email

parallel –colsep ‘,’ ‘echo {1} is {2} years old’ :::: data.csv

并行统计每 1000 行的单词数

cat bigfile.txt | parallel –pipe –block 1000K wc -w

处理以结尾的 XML 记录

cat data.xml | parallel –recend ‘‘ –pipe ‘xsltproc transform.xsl -‘

在 server1 上并行压缩文件

parallel -S server1 gzip ::: file1.txt file2.txt

~/.parallel/sshloginfile 内容：

server1

server2

8/server3 # 8 核服务器

parallel –slf sshloginfile ‘hostname’ ::: {1..100}

将本地文件传到远程处理，结果返回到本地并清理临时文件

parallel -S server1 –trc {}.processed ‘process {} > {}.processed’ ::: input_*.dat

并行下载 URL 列表中的文件

cat urls.txt | parallel -j 8 wget {}

在集群上并行处理日志文件

parallel –sshloginfile nodes.txt ‘grep “ERROR” {} > {}.errors’ ::: *.log

并行运行 Python 脚本，参数来自参数组合

parallel -j 4 ‘python simulate.py {1} {2}’ ::: 0.1 0.5 1.0 ::: 10 20 30

parallel –dry-run ‘echo Processing {}’ ::: input_*.txt


parallel –progress ‘convert {} {.}.png’ ::: *.jpg
