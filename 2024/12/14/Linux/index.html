Linux系统目录结构

  1. /bin：
    bin 是 Binaries (二进制文件) 的缩写, 这个目录存放着最经常使用的命令。

  2. /boot：
    这里存放的是启动 Linux 时使用的一些核心文件，包括一些连接文件以及镜像文件。

  3. /dev ：
    dev 是 Device(设备) 的缩写, 该目录下存放的是 Linux 的外部设备，在 Linux 中访问设备的方式和访问文件的方式是相同的。

  4. /etc：
    etc 是 Etcetera(等等) 的缩写,这个目录用来存放所有的系统管理所需要的配置文件和子目录。

  5. /home：
    用户的主目录，在 Linux 中，每个用户都有一个自己的目录，一般该目录名是以用户的账号命名的。

  6. /lib：
    lib 是 Library(库) 的缩写这个目录里存放着系统最基本的动态连接共享库，其作用类似于 Windows 里的 DLL 文件。几乎所有的应用程序都需要用到这些共享库。

  7. /lost+found：
    这个目录一般情况下是空的，当系统非法关机后，这里就存放了一些文件。

  8. /media：
    linux 系统会自动识别一些设备，例如U盘、光驱等等，当识别后，Linux 会把识别的设备挂载到这个目录下。

  9. /mnt：
    系统提供该目录是为了让用户临时挂载别的文件系统的，我们可以将光驱挂载在 /mnt/ 上，然后进入该目录就可以查看光驱里的内容了。

  10. /opt：
    opt 是 optional(可选) 的缩写，这是给主机额外安装软件所摆放的目录。比如你安装一个ORACLE数据库则就可以放到这个目录下。默认是空的。

  11. /proc：
    proc 是 Processes(进程) 的缩写，/proc 是一种伪文件系统（也即虚拟文件系统），存储的是当前内核运行状态的一系列特殊文件，这个目录是一个虚拟的目录，它是系统内存的映射，我们可以通过直接访问这个目录来获取系统信息。

  12. /root：
    该目录为系统管理员，也称作超级权限者的用户主目录。

  13. /sbin：
    s 就是 Super User 的意思，是 Superuser Binaries (超级用户的二进制文件) 的缩写，这里存放的是系统管理员使用的系统管理程序。

  14. /selinux：
    这个目录是 Redhat/CentOS 所特有的目录，Selinux 是一个安全机制，类似于 windows 的防火墙，但是这套机制比较复杂，这个目录就是存放selinux相关的文件的。

  15. /srv：
    该目录存放一些服务启动之后需要提取的数据。

  16. /sys：

    这是 Linux2.6 内核的一个很大的变化。该目录下安装了 2.6 内核中新出现的一个文件系统 sysfs 。

    sysfs 文件系统集成了下面3种文件系统的信息：针对进程信息的 proc 文件系统、针对设备的 devfs 文件系统以及针对伪终端的 devpts 文件系统。

    该文件系统是内核设备树的一个直观反映。

    当一个内核对象被创建的时候，对应的文件和目录也在内核对象子系统中被创建。

  17. /tmp：
    tmp 是 temporary(临时) 的缩写这个目录是用来存放一些临时文件的。

  18. /usr：
    usr 是 unix shared resources(共享资源) 的缩写，这是一个非常重要的目录，用户的很多应用程序和文件都放在这个目录下，类似于 windows 下的 program files 目录。

  19. /usr/bin：
    系统用户使用的应用程序。

  20. /usr/sbin：
    超级用户使用的比较高级的管理程序和系统守护程序。

  21. /usr/src：
    内核源代码默认的放置目录。

  22. /var：
    var 是 variable(变量) 的缩写，这个目录中存放着在不断扩充着的东西，我们习惯将那些经常被修改的目录放在这个目录下。包括各种日志文件。

  23. /run：
    是一个临时文件系统，存储系统启动以来的信息。当系统重启时，这个目录下的文件应该被删掉或清除。如果你的系统上有 /var/run 目录，应该让它指向 run。

Linux常用命令

  1. ls（英文全拼：list files）: 列出目录及文件名
  2. cd（英文全拼：change directory）：切换目录
  3. pwd（英文全拼：print work directory）：显示目前的目录
  4. mkdir（英文全拼：make directory）：创建一个新的目录
  5. rmdir（英文全拼：remove directory）：删除一个空的目录
  6. cp（英文全拼：copy file）: 复制文件或目录
  7. rm（英文全拼：remove）: 删除文件或目录
  8. mv（英文全拼：move file）: 移动文件与目录，或修改文件与目录的名称
  9. 忘记了命令什么用用man [command]来查看相应命令的帮助文档。注意空格不可以少！
  10. sort：排序

cp (复制文件或目录)

语法

选项与参数

  1. **-a：**相当于 -pdr 的意思，至于 pdr 请参考下列说明；(常用)
  2. **-d：**若来源档为链接档的属性(link file)，则复制链接档属性而非文件本身；
  3. **-f：**为强制(force)的意思，若目标文件已经存在且无法开启，则移除后再尝试一次；
  4. **-i：**若目标档(destination)已经存在时，在覆盖时会先询问动作的进行(常用)
  5. **-l：**进行硬式链接(hard link)的链接档创建，而非复制文件本身；
  6. **-p：**连同文件的属性一起复制过去，而非使用默认属性(备份常用)；
  7. **-r：**递归持续复制，用于目录的复制行为；(常用)
  8. **-s：**复制成为符号链接档 (symbolic link)，亦即『捷径』文件；
  9. **-u：**若 destination 比 source 旧才升级 destination ！

sort（排序）

参数

  • 默认情况下，sort 会按字母顺序（ASCII 顺序）对文件排序
  • -n 选项可以按数字顺序排序。10 会排在 2 后面，而不是 1 后面
  • -r 选项可以进行反向排序，即降序排序
  • -k 选项指定按哪一列排序
  • -f 选项可以忽略大小写（例如，a 和 A 被视为相同的字符）
  • -u 选项，可以在排序的同时去掉重复的行
  • -t 选项指定分隔符
  • -s 选项可以进行稳定排序，保持相等元素的原始顺序。（默认的 sort 是不稳定的，即如果两个元素相等，它们的顺序可能会被打乱）

Linux文本编辑器 – VIM

在 Linux 中，就像任何类 Unix 操作系统，一切都只是文件。而 Unix 哲学指出，配置文件必须是人类可读和可编辑的。在几乎所有的情况下，它们只是纯文本。VIM分为普通（命令）模式与插入模式，使用vim打开文件后进入的是普通模式，按i进入插入模式，以及 <ESCAPE>返回到普通模式。

  • h - 向左移动
  • j - 向下移动
  • k - 向上移动l - 右移
  • i - 进入插入模式
  • o - 在光标下插入一行并进入插入模式<ESCAPE> - 退出插入模式
  • x - 删除光标下的符号
  • dd - 删除一行
  • :%d -删除所有内容 （%：表示整个文件范围，d：表示删除）
  • :13,57d - 删除15到57行
  • :wq - 将更改写入文件并退出。是的，没错，这是一个冒号，后面跟着 wq 和<ENTER>。
  • :q! - 不要对文件进行更改并退出。
  • G: 光标移动到最下面

Bash：Shell、 .profile 、 .bashrc 、 .bash_history

Shell：当使用 CLI（命令行界面）来使用 Linux 时，你正在与一个名为 shell 的程序进行交互。所有你输入的都传递给 shell，它解释你输入的内容，执行参数扩展，并为你执行程序。我们将使用的 Shell 称为 Bash，它代表 Bourne Again Shell。 Zsh（Z Shell）比 Bash 更现代，用户体验更好。

.profile 是一个用户级配置文件，用于定义用户的环境变量和 Shell 的初始化设置。它主要适用于 登录 Shell（用户通过终端或 SSH 登录时运行的 Shell）。

.bashrc 是专为 非登录 Shell（如直接打开一个终端窗口）设计的初始化文件。它主要用来定义别名、函数、Shell 提示符（PS1）等，常用于定制用户的命令行体验。

文件名作用触发条件常见用途
Shell用户与操作系统交互的工具用户输入命令时执行命令、脚本
.profile配置登录 Shell 的环境变量和初始化设置用户登录系统时（如 SSH）设置环境变量、加载 .bashrc
.bashrc配置非登录 Shell 的别名、函数等打开新终端或运行 Bash 时定义别名、函数、Shell 提示符
.bash_history保存用户输入过的 Bash 命令历史每次运行 Bash 命令后命令历史记录与回溯

Bash：环境变量，env，set，export

命令作用是否影响子进程备注
env显示环境变量，或者临时设置环境变量执行命令只对该命令有效不会永久改变环境变量
set显示当前 Shell 中所有变量（包括环境变量、局部变量、Shell 函数等）不直接影响子进程还可用于修改 Shell 的行为
export将变量导出为环境变量，供子进程使用会影响子进程需要明确指定变量进行导出

Bash：重定向， stdin ， stdout ， stderr ， < ， > ，>> ， | ， tee ， pv

重定向 是一种功能，可以控制输入和输出的流向。默认情况下，程序的输入（stdin）来自键盘，输出（stdout）和错误信息（stderr）显示在终端。但通过重定向，我们可以改变这些流的方向。

  • >：重定向。标准输出将命令的输出重定向到文件，覆盖文件内容。

  • >>：追加标准输出。将命令的输出重定向到文件，追加到文件末尾。

  • <：重定向标准输入。将文件内容作为命令的输入。

  • 2>：重定向标准错误。将错误信息输出到文件，覆盖文件内容。

  • 2>>：追加标准错误。将错误信息追加到文件末尾。

  • &>：同时重定向标准输出和错误。将标准输出和错误同时重定向到文件。

  • |：管道符。将一个命令的输出作为另一个命令的输入。

  • tee：同时输出到终端和文件。tee 命令将标准输入的数据输出到 终端 和 文件。

  • tee -a：同时输出到终端和文件。tee 命令将标准输入的数据输出到 终端 和 文件。（不覆盖文件）

note

pv（Pipe Viewer） 是一个非常实用的命令行工具，用于在处理数据流时显示进度信息。它可以监控数据通过管道的流速、传输量以及预计完成的时间。

note

流的分类：

  • stdin**（标准输入）：**输入流，默认从键盘读取，文件描述符为 0。
  • stdout**（标准输出）：**输出流，默认输出到终端，文件描述符为 1。
  • stderr**（标准错误）：**错误流，默认输出到终端，文件描述符为 2。

通配符（Wildcards）

  1. \*（星号）：
    • 匹配任意长度的任意字符（包括空字符）。
    • 示例：
      • *.txt：匹配所有以 .txt 结尾的文件（如 file.txt、note.txt）。
      • file*：匹配所有以 file 开头的文件（如 file1、file_backup）。
  2. ?（问号）：
    • 匹配单个任意字符。
    • 示例：
      • file?.txt：匹配 file1.txt、fileA.txt，但不匹配 file10.txt。
  3. [...]（方括号）：
    • 匹配方括号内的任意一个字符。
    • 示例：
      • file[123].txt：匹配 file1.txt、file2.txt、file3.txt。
      • file[a-c].txt：匹配 filea.txt、fileb.txt、filec.txt。

head，tail，awk，grep，sed

  • head：显示文件的开头内容

    • -n [行数]：显示文件的前 n 行，默认显示 10 行。
    • -c [字节数]：显示文件的前 n 个字节。

  • tail：显示文件的结尾内容

    • -n [行数]：显示文件的最后 n 行，默认显示 10 行。
    • -c [字节数]：显示文件的最后 n 个字节。
    • -f：实时跟踪文件内容的变化（多用于监控日志）。

  • grep：文本搜索工具

    • -i：忽略大小写。
    • -v：反向匹配（排除匹配内容）。
    • -n：显示匹配行的行号。
    • -r：递归搜索目录中的所有文件。
    • -E：启用正则表达式（也可直接使用 egrep 命令）。

  • awk：高级文本处理工具

    • awk '条件 { 动作 }' 文件
    • 正则表达式融入

  • sed：流编辑器

    • sed '操作' 文件
    • s：替换操作。
    • d：删除匹配的行。
    • i：在指定行前插入内容。
    • a：在指定行后添加内容。

命令功能描述示例备注
jobs显示当前 Shell 中的后台任务列表jobs显示任务编号、状态（如运行或暂停）及命令名称

将任务挂后台，并且标准输出和标准错误重定向，不显示在终端中

command > /dev/null 2>&1 &

  • command：你想要执行的命令。
  • >：将标准输出（stdout）重定向。
  • /dev/null：一个特殊的设备，所有写入它的数据都会被丢弃，起到“丢弃输出”的作用。
  • 2>&1：将标准错误输出（stderr，文件描述符 2）重定向到标准输出（stdout，文件描述符 1）。这样，所有的输出（标准输出和标准错误输出）都会被丢弃。
  • &：将命令放到后台执行。

command &>/dev/null &

  • command：你想执行的命令。
  • &>：这是将标准输出和标准错误都重定向的简写。
  • /dev/null：丢弃输出。
  • &：将命令放到后台运行。

文件系统：权限， chown ， chmod

文件权限（File Permissions）每个文件和目录都有三种权限：

  • 读权限（r）：允许查看文件内容。
  • 写权限（w）：允许修改文件内容。
  • 执行权限（x）：允许执行文件（对于目录，表示可以进入该目录）。

每个文件的权限由三部分组成：

  • 用户（User）：文件的拥有者。
  • 组（Group）：文件所属的用户组。
  • 其他（Others）：所有其他用户。

符号方式：使用字母 r、w、x 表示读、写、执行权限，使用 +、- 来添加或去除权限。
